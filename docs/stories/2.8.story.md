## Story

**Como** aluno/instrutor do workshop "Living Lab Azure-Native",
**eu quero** ampliar o chatbot da Fase 5 com 4 novas tools MCP read-only (`consultar_partidas`, `consultar_classificacao`, `consultar_time`, `consultar_estadio`) no McpServer existente,
**para que** o chat passe a "explorar todo o sistema da Copa" — respondendo perguntas como "Quando o Brasil joga?", "Como está o grupo A?", "Quem está na semifinal?" e "Me fala do Maracanã?" consultando o SQL real, sem alucinação e sem abrir vetor de escrita no banco.

## Acceptance Criteria

1. **AC-1 — `consultar_partidas` implementada (tool-âncora, ADE-006 A.1):** método estático em `FifaTickerTools.cs` (arquivo real do McpServer; classe `FifaTicketTools`) — OU classe homóloga nomeada `FifaAgenticTools.cs`, a critério do @dev. **Nota PO (anti-alucinação):** o arquivo real é `Tools/FifaTickerTools.cs` (com "Ticker", typo herdado), a classe é `FifaTicketTools`, e o arquivo de testes é `FifaTicketToolsTests.cs`. O `fase` deve mapear via `MapFaseToStage` que **delega** a `MapRodadaToStage` (existente) e adiciona apenas o caso "grupos" → `'Fase de Grupos'` — não duplicar os mapeamentos de mata-mata; o teste existente `BracketStageMappingTests` sobre `MapRodadaToStage` (que assere `"grupo A" → null`) deve permanecer verde. com `[McpServerTool(Name = "consultar_partidas", ReadOnly = true)]` e `[Description]` em PT-BR: `"Consulta partidas da Copa 2026 com filtros flexíveis. Use para perguntas como 'jogos do Brasil', 'jogos nas oitavas', 'jogos no Maracanã', 'jogos do grupo C' ou 'jogos do dia 15/06'. Retorna o placar quando o jogo já foi disputado."` — Parâmetros todos opcionais com `= null` (SDK 1.4.0 marca nullable sem default como `required` — bug conhecido de S2.5): `time` (string), `fase` (string), `estadio` (string), `grupo` (string), `data` (string ISO YYYY-MM-DD), `apenasComResultado` (bool). Retorna `IReadOnlyList<MatchResult>` com DTO em `McpQueryResults.cs` (`[JsonPropertyName]` em PT-BR). Fontes SQL: `matches` LEFT JOIN `teams` (home/away) LEFT JOIN `stadiums`. Filtro de `fase` mapeia linguagem natural ("grupos", "oitavas", "quartas", "semifinal", "final", "round of 32") para `matches.stage`; "grupos" → `'Fase de Grupos'` (com acento — valor real da migration `2026-05-08-group-stage-72.sql`). `apenasComResultado=true` → `m.home_score IS NOT NULL`. Times NULL no mata-mata → `COALESCE(ht.name,'A definir')` (padrão de `consultar_bracket`).

2. **AC-2 — `consultar_classificacao` implementada (ADE-006 A.2):** `[McpServerTool(Name = "consultar_classificacao", ReadOnly = true)]`, `[Description]`: `"Consulta a classificação (tabela de pontos) de um grupo da fase de grupos da Copa 2026. Calculada a partir dos resultados dos jogos do grupo."` — Parâmetro `grupo` (string, obrigatório — sem `= null`). Retorna `IReadOnlyList<StandingRow>` com `{ posicao, time, jogos, vitorias, empates, derrotas, golsPro, golsContra, saldo, pontos }`. **Não existe tabela `standings`** — a classificação é calculada por agregação: `UNION ALL` expandindo cada partida de `matches` onde `group_name = @grupo AND stage = 'Fase de Grupos' AND home_score IS NOT NULL` em duas linhas (mandante/visitante), somando 3/1/0 pontos, GROUP BY `team_id`, ORDER BY `pontos DESC, saldo DESC, golsPro DESC`, JOIN `teams` para o nome. **Decisão PO (resolve ambiguidade do @sm):** antes de qualquer jogo do grupo ser disputado (`home_score IS NULL` em todas as partidas), a tool retorna **lista vazia** (comportamento exigido — opção (a)). NÃO incluir todos os times com 0 pontos via segunda query (opção (b) descartada): a agregação verdadeira não produz linhas sem jogos, e o `[Description]` da tool + a resposta do chatbot já informam "ainda sem jogos disputados" (AC-10). O `[Description]` deve avisar dessa condição.

3. **AC-3 — `consultar_time` implementada (ADE-006 A.3):** `[McpServerTool(Name = "consultar_time", ReadOnly = true)]`, `[Description]`: `"Consulta informações de uma seleção da Copa 2026 (grupo, confederação, ranking FIFA, código)."` — Parâmetro `nome` (string, obrigatório). Retorna `TeamResult` com `{ encontrado, nome, codigo, grupo, confederacao, rankingFifa, bandeira }`. Fonte SQL: `teams` (`name`, `code`, `group_name`, `confederation`, `fifa_ranking`, `flag`). Match: `name LIKE @nome OR code = @nome` (case-insensitive via `LOWER()`/`COLLATE` ou LIKE com param em uppercase no app).

4. **AC-4 — `consultar_estadio` implementada (ADE-006 A.4):** `[McpServerTool(Name = "consultar_estadio", ReadOnly = true)]`, `[Description]`: `"Consulta informações de um estádio/sede da Copa 2026 (cidade, país, capacidade, descrição)."` — Parâmetro `nome` (string, obrigatório — nome do estádio OU cidade).   - [ ] 3.2 `ConsultarClassificacaoAsync`: query de agregação via UNION ALL. Estrutura base (sem tabela `standings` — não existe):
    ```sql
    WITH PartidosDoGrupo AS (
        SELECT
            m.home_team_id AS team_id,
            m.home_score   AS gols_marcados,
            m.away_score   AS gols_sofridos,
            CASE WHEN m.home_score > m.away_score THEN 3
                 WHEN m.home_score = m.away_score THEN 1
                 ELSE 0 END AS pontos
        FROM dbo.matches m
        WHERE m.group_name = @Grupo
          AND m.stage = 'Fase de Grupos'
          AND m.home_score IS NOT NULL
        UNION ALL
        SELECT
            m.away_team_id,
            m.away_score,
            m.home_score,
            CASE WHEN m.away_score > m.home_score THEN 3
                 WHEN m.away_score = m.home_score THEN 1
                 ELSE 0 END
        FROM dbo.matches m
        WHERE m.group_name = @Grupo
          AND m.stage = 'Fase de Grupos'
          AND m.home_score IS NOT NULL
    )
    SELECT
        ROW_NUMBER() OVER (ORDER BY SUM(p.pontos) DESC, SUM(p.gols_marcados - p.gols_sofridos) DESC, SUM(p.gols_marcados) DESC) AS Posicao,
        t.name          AS Time,
        COUNT(*)        AS Jogos,
        SUM(CASE WHEN p.pontos = 3 THEN 1 ELSE 0 END) AS Vitorias,
        SUM(CASE WHEN p.pontos = 1 THEN 1 ELSE 0 END) AS Empates,
        SUM(CASE WHEN p.pontos = 0 THEN 1 ELSE 0 END) AS Derrotas,
        SUM(p.gols_marcados)                           AS GolsPro,
        SUM(p.gols_sofridos)                           AS GolsContra,
        SUM(p.gols_marcados - p.gols_sofridos)         AS Saldo,
        SUM(p.pontos)                                  AS Pontos
    FROM PartidosDoGrupo p
    JOIN dbo.teams t ON t.id = p.team_id
    GROUP BY t.id, t.name
    ORDER BY Pontos DESC, Saldo DESC, GolsPro DESC;
    ```
    Nota (decisão PO — não opcional): se nenhum jogo foi disputado no grupo, a query retorna **lista vazia** (comportamento EXIGIDO — AC-2, opção (a)). ### Gotchas críticos de implementação

**Gotcha 1 — `'Fase de Grupos'` tem acento e espaços (valores de `matches.stage`):**
Os valores reais de `matches.stage` são:
- Fase de grupos → `'Fase de Grupos'` (inserido por `2026-05-08-group-stage-72.sql`)
- Mata-mata: `'round_of_32'`, `'round_of_16'`, `'quarter_final'`, `'semi_final'`, `'third_place'`, `'final'` (inseridos por `2026-05-07-knockout-matches.sql`)

`MapFaseToStage` deve mapear "grupos" / "grupo" / "fase de grupos" → `'Fase de Grupos'`. Os mapeamentos de mata-mata já existem em `MapRodadaToStage` — reutilizar (ou fazer `MapFaseToStage` chamar `MapRodadaToStage` como fallback, adicionando só o caso "grupos").

**Gotcha 2 — `matches.group_name` é NULL no mata-mata:**
`group_name` só é preenchido na fase de grupos (valores A–L). Para `consultar_partidas` com filtro de `fase = "grupos"`, o filtro `stage = 'Fase de Grupos'` já é suficiente — não filtrar por `group_name IS NOT NULL` separadamente (redundante e potencialmente confuso).

**Gotcha 3 — Standings não existem como tabela:**
`consultar_classificacao` **agrega** resultados de `matches`. Não existe tabela `standings`, `classificacao` ou similar no schema real. A query UNION ALL (Task 3.2) é a implementação correta. Se o @dev consultar o schema SQL e não encontrar `standings`, é esperado.

**Gotcha 4 — Build no ACR requer limpar `obj/bin`:**
O `az acr build` envia o contexto do diretório para o serviço de build do Azure. Se `obj/bin` com binários Windows estiverem presentes, o build Linux do ACR pode falhar com erros de plataforma cruzada. Sempre remover antes do `az acr build src/Fifa2026.V2.McpServer/ ...` (Task 6.1).

**Gotcha 5 — gateway cache é GET-only (já resolvido):**
O fix do commit `e5451ce` garantiu que o output cache do gateway só se aplica a GET. Chamadas POST /mcp (tools/call) não são cacheadas. Não é necessário nenhuma ação adicional — apenas documentar no smoke que o comportamento é esperado.

**Gotcha 6 — modelo LLM mudou de `gemini-2.0-flash` para `gemini-2.5-flash`:**
O fix do commit `9c9a549` atualizou o modelo no backend (proxy LLM). Confirmar se `Lovable/World Cup Tickets Hub/src/lib/llm/gemini.ts` também foi atualizado (Task 7.1). Se ainda referenciar o modelo antigo, corrigir para `gemini-2.5-flash`.

### Estratégia de classificação calculada (anti-alucinação ADE-006 Art. IV)

A tabela `standings` **não existe** no schema real do projeto. A classificação do grupo deve ser **calculada** por agregação dos jogos de `matches` onde `group_name = @grupo AND stage = 'Fase de Grupos' AND home_score IS NOT NULL`. A query UNION ALL expande cada partida em dois registros (perspectiva do mandante e do visitante), soma pontos (3/1/0) e agrega por time. Antes dos jogos serem disputados, a query retorna lista vazia (ou lista com 0 pontos se o @dev incluir os times do grupo via segunda query — ver Task 3.2 nota). Ambos os comportamentos são corretos e devem ser documentados no `[Description]` da tool.
