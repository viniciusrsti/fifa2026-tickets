## Dev Notes

### CORRECAO CRITICA: Gateway YARP e o No Zero (nao APIM)

O draft original desta story (v0.1) referenciava **APIM** como o primeiro nó do fluxo e como o componente que injeta `X-Correlation-ID`. Isso estava ERRADO. A arquitetura real (pós re-escopo ADE-004, confirmada na implementação de F2) é:

**APIM não existe no EPIC-002.** O nó zero do fluxo é o **Gateway YARP** (`src/Fifa2026.V2.Gateway/`), um projeto ASP.NET Core + `Yarp.ReverseProxy` hospedado em Container App por aluno.

O `X-Correlation-ID` é injetado pelo gateway em **`src/Fifa2026.V2.Gateway/Program.cs`** via `AddRequestTransform`:
```csharp
// Trecho real confirmado:
var incoming = transformContext.HttpContext.Request.Headers[CorrelationHeader].ToString();
var correlationId = string.IsNullOrWhiteSpace(incoming)
    ? Guid.NewGuid().ToString()
    : incoming;
transformContext.ProxyRequest.Headers.TryAddWithoutValidation(CorrelationHeader, correlationId);
// Devolve o mesmo correlationId ao cliente:
transformContext.HttpContext.Response.Headers[CorrelationHeader] = correlationId;
```

Portanto, o Flow Visualizer deve mostrar **6 nós** na seguinte ordem:

```
[0] Gateway YARP     ← NÓ ZERO: injeta X-Correlation-ID, valida JWT (F3)
[1] Function Entry   ← PurchaseEntryFunction: publica no Service Bus
[2] Service Bus      ← tickets-purchase: mensagem com CorrelationId em ApplicationProperties
[3] Function Consumer ← PurchaseConsumerFunction: grava no SQL, dispara n8n (F4)
[4] n8n              ← Workflow post-purchase-notification: log estruturado
[5] SQL              ← purchases.correlation_id gravado
```

### Troubleshooting conhecido

| Sintoma | Causa provável | Solução |
|---|---|---|
| Bolinha não aparece para um nó | Trace não chegou ao App Insights | Verificar se o componente tem App Insights configurado; aguardar latência de ingestão (até 2min) |
| SignalR não conecta | WebSocket bloqueado | Ativar fallback polling (2s); verificar CORS no SignalR Service |
| Nó "Gateway YARP" ausente no diagrama | Referência antiga ao APIM | Confirmar que o diagrama usa os 6 nós corretos (Gateway YARP, não APIM) |
| correlationId não aparece no App Insights | BeginScope não configurado em algum componente | Verificar ILogger.BeginScope com CorrelationId em todas as Functions |
| Latência > 30s para bolinha completar | App Insights ingestão lenta | Usar near-real-time query ou pré-popular eventos via polling |
